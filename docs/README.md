# ドキュメント一覧

このディレクトリには、Coast FIRE Next.js アプリケーションの開発者向けドキュメントが含まれています。

## 📚 ドキュメント

### [開発者ガイド](./DEVELOPER_GUIDE.md)

開発環境のセットアップから、コーディング規約、テスト、デバッグまで、開発に必要な情報を網羅したガイドです。

**主な内容:**

- 開発環境のセットアップ手順
- プロジェクト構造の詳細説明
- コーディング規約（TypeScript、React、Tailwind CSS）
- テスト戦略とベストプラクティス
- デバッグ方法
- 開発ワークフロー
- トラブルシューティング

**対象読者:** 初めて本プロジェクトに参加する開発者、プロジェクト全体の流れを理解したい方

---

### [アーキテクチャ設計書](./ARCHITECTURE.md)

システムの設計思想、アーキテクチャパターン、技術選定の理由などを解説したドキュメントです。

**主な内容:**

- システム概要と技術選定の理由
- アーキテクチャの原則
- レイヤー構造（表示層、ビジネスロジック層、型定義層）
- コンポーネント設計パターン
- データフローと状態管理
- ルーティング構造
- スタイリング戦略
- パフォーマンス最適化
- セキュリティ考慮事項

**対象読者:** システムの全体像を理解したい方、設計に関わる方、技術選定の背景を知りたい方

---

### [API リファレンス](./API_REFERENCE.md)

関数、型定義、コンポーネントの詳細な仕様を記載したリファレンスドキュメントです。

**主な内容:**

- 計算ロジック API（`calculateCoastFire`、フォーマット関数）
- 型定義（`CoastFireInput`、`CoastFireResult`、`YearlyData`）
- コンポーネント API（Props、使用例）
- ユーティリティ関数
- Context API（`useDarkMode`）
- エラーハンドリング

**対象読者:** 具体的な API の使い方を知りたい方、実装の詳細を確認したい方

---

## 📖 読む順番の推奨

### 初めての方

1. **[開発者ガイド](./DEVELOPER_GUIDE.md)** - まずはここから開始
2. **[アーキテクチャ設計書](./ARCHITECTURE.md)** - システムの全体像を理解
3. **[API リファレンス](./API_REFERENCE.md)** - 必要に応じて参照

### 既存の開発者

- 日常的な開発: **[API リファレンス](./API_REFERENCE.md)**
- 設計変更や新機能追加: **[アーキテクチャ設計書](./ARCHITECTURE.md)**
